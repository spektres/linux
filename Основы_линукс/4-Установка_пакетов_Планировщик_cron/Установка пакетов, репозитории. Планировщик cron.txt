=================APT=================
sudo apt install -y nginx - установка без подтверждения
sudo apt remove nginx - удаляет только один пакет(без зависимостей)
sudo apt autoremove nginx - удаляет зависимости(конфигурация остается)
sudo apt purge *** - удаляет все полностью, но *** нужно указать зависимости(список всех пакеов, кот. были установ.)

================DPKG===================
dpkg - для самостоятельной установки(проверяем версию дистрибутива и программы)
wget *** - скачиваем по ссылке http, затем:
dpkg -i *** - установка скаченной программы
Возникают проблемы с завсимостями, необходимо исправить сломанные пакеты:
sudo apt -f install
! ПРИ установке новой версии софта, нужно снова его скачать и проделать те же действия(зависимости уже устаовленны),
т.к. это автономная установка.
Для решения проблеммы с обновлением и установки пакета максимально корректно нужно 
смотреть инструкцию (у каждого разработчика есть инструкция для его конкретного репозитория)
Наприер для vbox:
deb [arch=amd64 signed-by=/usr/share/keyrings/oracle-virtualbox-2016.gpg] https://download.virtualbox.org/virtualbox/debian <mydist> contrib
<mydist> - указатель на версию дистрибутива. У нас jammy.
Теперь понадобится дириктория с конфигами cd /etc/apt/source.list.d
Здесь сздаем новый файл, в котором будет прописан репозиторий:
cat > vb.list
вставляем то, что было в буфере
deb [arch=amd64 signed-by=/usr/share/keyrings/oracle-virtualbox-2016.gpg] https://download.virtualbox.org/virtualbox/debian jammy contrib
проверяем: apt update
НЕТ КЛЮЧА
читаем дальше инсрукцию на сайте:
wget -O- https://www.virtualbox.org/download/oracle_vbox_2016.asc | sudo gpg --dearmor --yes --output /usr/share/keyrings/oracle-virtualbox-2016.gpg
эта команда скачивает и устанавливает ключ
проверяем: apt update
Если все ок, то при появлении новой версии, по команде apt upgrade получим ее.

====================SNAP========================
В этих пакетах нет проблем с зависимостями.
